## API Endpoints

The following API endpoints are available:

*   `POST api/v2/login`
*   `POST api/v2/register`
*   `POST api/v2/re-create-profile`
*   `GET api/v2/me`
*   `POST api/v2/forgot-password`
*   `POST api/v2/reset-password/:token`
*   `POST api/v2/media/upload`
*   `POST api/v2/media/upload-multiple`
*   `POST api/v2/media/upload-private`
*   `POST api/v2/media/upload-multiple-private`
*   `GET api/v2/media/:mediaId/sign`
*   `GET api/v2/groups`
*   `GET api/v2/avatar3d`
    *   **Filter by user ID:** `GET api/v2/avatar3d?user=userId`
*   `POST api/v2/avatar3d`
    *   **Request body:** `{ name, avatar2D, platform }`
*   `PUT api/v2/avatar3d/:id`
    *   **Request body:** `{ name, avatar2D, platform }`
*   `DELETE api/v2/avatar3d/:id`
*   `GET api/v2/ambassador-codes`
*   `GET api/v2/emojis`

To test these endpoints, you can use a tool like Postman or `curl`. For example, to test the `POST api/v2/login` endpoint, you can use the following `curl` command:

```
curl -X POST -H "Content-Type: application/json" -d '{"email": "test@example.com", "password": "password"}' http://localhost:2222/api/v2/login
```

Note that you need to replace `test@example.com` and `password` with the actual email and password. Also, the backend is running on port 2222, so you need to use `http://localhost:2222` as the base URL.
